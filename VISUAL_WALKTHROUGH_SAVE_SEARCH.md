## Data Flow Diagram

```
┌─────────────────────┐
│  Generated App      │
│  (Single HTML File) │
└──────────┬──────────┘
           │
           ├─ SAVE Feature
           │  ├─ User enters: name, location, content
           │  └─ POST /api/hub/app-submission-save
           │     └─ Saves to Supabase
           │
           └─ SEARCH Feature
              ├─ User enters: name, location, type
              ├─ GET /api/hub/app-submission-search
              │  └─ Returns matching submissions
              └─ User clicks result
                 ├─ GET /api/hub/app-submission?id=...
                 │  └─ Returns full content
                 └─ Display in UI
```

## Complete Request/Response Examples

### 1. SAVE REQUEST
```
POST /api/hub/app-submission-save HTTP/1.1
Content-Type: application/json

{
  "appName": "GrandLetters",
  "appIdea": "Write & share letters from grandparents to grandkids",
  "name": "Margaret Johnson",
  "location": "Portland, Oregon",
  "submissionType": "letter",
  "content": "My dearest grandchildren, I hope this letter finds you well..."
}

RESPONSE:
HTTP/1.1 200 OK
{
  "success": true,
  "id": "abc123def456",
  "message": "Submission saved successfully"
}
```

✅ **Other users** → Can search and find same data
